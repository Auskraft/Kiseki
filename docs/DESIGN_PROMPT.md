# Промт для генерации дизайна — Kiseki (итерация 1: медиа)

> Готовый текст для вставки в Claude (или другой дизайн-инструмент). Самодостаточен — не требует остальных документов. Основан на [TECH_DESIGN.md](TECH_DESIGN.md).

---

Ты — продуктовый UI/UX-дизайнер. Спроектируй визуальный дизайн и дизайн-систему мобильного приложения **Kiseki** — персональной офлайн-картотеки. Нужен дизайн **итерации 1** (домен «медиа»: фильмы, сериалы, дорамы, аниме). Результат должен быть реализуем во Flutter (Material 3).

## О приложении
- «Умная записная книжка» / личная картотека. Полностью офлайн, без аккаунта; данные локальны, резервная копия — в облако. Тон — личный, уютный, спокойный; визуально-ориентированный (постеры/обложки).
- Язык интерфейса — **русский**. Платформы — **Android и iOS** (mobile-first, но вёрстка тянется до планшета).
- Это первый из многих доменов (дальше — еда, рестораны, вейп). Дизайн-система и базовые компоненты доменно-нейтральны; медиа лишь наполняет их.

## Что отдать на выходе
1. **Дизайн-систему токенов** (см. требование ниже).
2. **High-fidelity макеты ключевых экранов** в light и dark + **2–3 примерных именованных темы** (как доказательство масштабируемости системы).
3. **Спецификацию переиспользуемых компонентов** с их состояниями.
4. Короткие заметки по адаптивности (телефон → планшет), доступности и тому, как длинный текст/переводы не ломают вёрстку.

## КЛЮЧЕВОЕ — дизайн-система под десятки тем
- Всё проектируй на **токенах**, без хардкода цветов/размеров. Система обязана поддерживать **несколько десятков тем**, у каждой — режим **light и dark**.
- Минимальный набор токенов: семантические цвета (surface / background / primary / secondary / on-* / outline / success / warning / error / accent оценки), типографика (роли текста), spacing-шкала, радиусы, тени/elevation, длительности анимаций.
- База — Material 3 (ColorScheme), а поверх — собственные семантические токены (под Flutter `ThemeExtension`). Любой компонент берёт значения только из токенов.
- Темы переключаются в рантайме. Покажи одну и ту же вёрстку в нескольких разных темах (контраст, читаемость сохраняются) — это критерий годности системы.

## i18n
- Весь текст — переменной длины (готовимся к переводам). Показывай реальные русские подписи, но компоненты должны переживать удлинение строк на +30–50% без поломки вёрстки.

## Экраны
1. **Главный — список/грид.** Переключатель грид↔список. Сверху горизонтальные «полки»: «Жду новые серии», «Смотрю сейчас» (+ опц. «Избранное»). Поиск-бар, вход в фильтры, FAB «+». Обновления мгновенные (офлайн).
2. **Детальная карточка.** Обложка; название + оригинальное название; год · страна · тип; чип статуса с быстрой сменой; крупная оценка «84 / 100» + шкала; блок прогресса серий (только для сериалов); три даты (начал / последний просмотр / досмотрел) с возможной приблизительностью; теги; личный отзыв; счётчик пересмотров; избранное; действия (редактировать, в корзину). До 3 фото (обычно одно).
3. **Создание/редактирование.** Прогрессивное раскрытие: обязательный минимум (тип, название, статус) → основное (оригинал, год, страна, обложка, теги, оценка) → прогресс серий (только для сериалов) → причина паузы/заброса (только в этих статусах) → даты → заметка. Поля показываются/скрываются реактивно.
4. **Корзина.** Мягко удалённые карточки; восстановление; окончательное удаление с подтверждением.
5. **Управление тегами.** Список тегов с цветом; переименование, слияние дублей, удаление.
6. **Настройки / Бэкап.** Подключение Яндекс.Диска, кнопка «Сделать бэкап», статус последнего успешного бэкапа, прогресс загрузки, восстановление с явным предупреждением «заменит локальные данные».
7. **Первый запуск / онбординг.** Пустое состояние + предложение восстановить из бэкапа.
8. **Состояния:** пустая картотека; пустой результат поиска; загрузка (скелетоны, не спиннеры); ошибка БД; потерянная картинка (плейсхолдер).

## Компоненты (с состояниями)
- **Карточка медиа** — двух видов: грид (постер 2:3 + название + бейдж оценки + иконка статуса + мини-прогресс серий + флаг избранного) и список-строка (thumbnail + название/оригинал + год + тип + статус + оценка + прогресс).
- **Бейдж оценки 0–100** — цвет по диапазону, НО число всегда продублировано текстом (не только цвет — для дальтоников). «Без оценки» = «—», не «0».
- **Чип статуса** — 5 значений: В планах / Смотрю / Просмотрено / На паузе / Заброшено. Узнаваемы не только цветом.
- **Прогресс серий** — «S2 · E5» и «12 / 24» + полоса (полоса только когда осмысленна: одно-сезонные с известным total).
- **Ввод оценки** — слайдер + числовое поле 0–100 (крупная зона нажатия).
- **Поле даты с гранулярностью** — переключатель «День / Месяц / Год», показ «12.03.2024» / «март 2024» / «2024», шорткаты «Сегодня» / «Не помню».
- **Чипы тегов** (с цветом) и селектор тегов.
- **Поиск-бар, лист фильтров (боттомшит), пресеты-фильтры.**
- **FAB, горизонтальные полки-карусели, плейсхолдер картинки по типу медиа.**

## Доменные UX-правила, влияющие на дизайн
- «Жду новые серии» = статус «На паузе» + причина «жду серии», и одновременно отдельная **полка** на главном. Это НЕ «заброшено» — визуально отделить ожидание от отказа.
- Оценка 0–100; `—` для неоценённого; неоценённые в сортировке уходят в хвост.
- Фильм vs сериал: блок «сезон/серия» есть только у сериалов; у фильма его нет.
- Причина «не досмотрел» (жду серии / скатился / не зашло / нет времени / другое) появляется только в статусах «На паузе» / «Заброшено».
- Картотека может быть большой (тысячи карточек) — грид плотный, но дышащий; thumbnail лёгкие.
- Тёмная тема важна (ведение/просмотр ночью).

## Ограничения и качество
- Зоны нажатия ≥ 48 dp; контраст по WCAG; смысл не передаётся только цветом; вёрстка переживает крупный системный шрифт (textScale до ~2.0).
- Офлайн-first: всё мгновенно; вместо спиннеров — скелетоны.
- Без перегруженности: чистый, современный «личный дневник», а не корпоративный дашборд.

## Формат ответа
Сначала — токены (списком, с семантикой), затем макеты экранов (light/dark + 2–3 темы), затем спецификации компонентов с состояниями, затем заметки по адаптивности / a11y / i18n. Если делаешь HTML-макеты — используй CSS-переменные как токены, чтобы наглядно показать переключение тем.
